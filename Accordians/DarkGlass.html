Developer Documentation

Deep-dive into our API, integrations, and configuration options.

  • All API requests require a Bearer token passed in the Authorization header. Keys are generated from your dashboard and scoped per environment. Production keys are never logged and rotate every 90 days by default. Use environment variables — never hardcode credentials in your codebase.
    REST API Secure OAuth2
  • Free tier allows 100 requests/minute per API key. Pro tier scales to 10,000 req/min with burst allowances. All responses include X-RateLimit-Remaining and X-RateLimit-Reset headers. Implement exponential backoff for 429 responses — aggressive retry loops will trigger temporary IP bans.
    Limits Headers
  • Register webhook endpoints via the dashboard or API. Each delivery includes an HMAC-SHA256 signature in the X-Signature header — always verify this before processing. Failed deliveries are retried with exponential backoff for up to 72 hours. Use our webhook tester to simulate events in staging.
    Real-time HMAC Retry
  • SDKs are available for Node.js, Python, Go, and Ruby. Install via your package manager of choice. The SDK handles auth, retries, pagination, and type safety automatically. Tree-shakeable builds for frontend use keep your bundle under 8kb gzipped.
    Node.js Python Go
  • All errors return a consistent JSON envelope with code, message, and trace_id fields. Use trace_id when contacting support. 4xx errors are client-side and will not resolve without changes. 5xx errors are transient — implement retries. Our status page at status.api.io shows real-time incident data.
    4xx · 5xx JSON
